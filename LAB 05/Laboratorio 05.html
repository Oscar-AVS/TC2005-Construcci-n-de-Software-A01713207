Preguntas del laboratorio

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet lo podemos entender como una infraestructura de redes globales que facilita el funcionamiento de la World Wide Web (WWW), la cual podemos entender como un servicio que nos permite acceder a la información que está disponible en Internet.

¿Cuáles son las partes de una URL?

Una URL (Uniform Resource Locator) está compuesta por 3 o 4 partes. El esquema que identifica el protocolo para acceder a Internet (HTTP o HTTPS). El nombre de host o dominio que identifica el servidor en donde se encuentra la página web. El camino que identifica la ruta específica para la página web. Y la cadena de consulta que se utiliza para proporcionar información como parámetros para realizar alguna búsqueda.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Su propósito es recolectar información de un servidor. HEAD: Su propósito es el mismo que GET, sin embargo solo recolecta los encabezados. POST: Su propósito es enviar datos a un servidor ya sea para crear un recurso o activarlo. PUT: Su propósito es reemplazar o actualizar un recurso por completo. PATCH: Su propósito es actualizar o modificar de manera parcial un recurso. DELETE: Su propósito es eliminar algún recurso.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debe utilizar POST, ya que permite enviar datos al servidor de manera más segura, esto es más importante cuando son datos usuario y contraseña.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza GET ya que el navegador solicita al servidor la información correspondiente a la página web indicada en la URL.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

El código 200 es un “OK”, lo cual significa que la solicitud fue exitosa. Por lo tanto, no ocurrió ningún error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí es responsabilidad del desarrollador ya que el código 404 significa que el recurso no fue encontrado ya sea porque las rutas son incorrectas o haya archivos eliminados, lo cual debe corregirse para garantizar el correcto funcionamiento del sitio.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si es responsabilidad del desarrollador ya que el error 500 indica un problema interno del servidor ya sea por errores en el código, configuraciones incorrectas o fallos en la base de datos, por lo que se debe solucionar para que el sitio funcione correctamente.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Que un atributo esté depreciado significa que ya no se recomienda su uso porque ha sido reemplazado por mejores prácticas o nuevas tecnologías Algunos elementos de HTML 4 que en HTML5 están depreciados Font, center, big, frame.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 incorpora nuevas etiquetas semánticas como <header>, <section>, <article> y <footer>, además de tener un soporte nativo para audio y video, mejoras en formularios, nuevas APIs y mejor compatibilidad con aplicaciones web modernas. HTML 4 era más limitado y en su mayoría dependía de plugins externos.

¿Qué componentes de estructura y estilo tiene una tabla?

Una tabla está compuesta por <table>, <tr> (filas), <td> (celdas), <th> (encabezados), <thead>, <tbody> y <tfoot>. Y de estilo, se puede modificar con CSS utilizando border, padding, background-color, text-align y width entre otros.

¿Cuáles son los principales controles de una forma HTML5?

Los principales controles son <input> (text, email, password, number, date, etc.), <textarea>, <select>, <option>, <button>, <label> y nuevos tipos como range, color y file.

¿Qué tanto soporte HTML5 tiene el navegador que utilizas?

Utilizo Google Chrome y de acuerdo con https://html5test.com/, tiene un alto nivel de compatibilidad con HTML5 teniendo 526/555 puntos.

¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de los sistemas de información incluye la planeación, análisis, diseño, implementación, operación y mantenimiento.

¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo de sistemas de información consiste en identificar el problema, analizar los requerimientos, diseñar la solución, programar el sistema, realizar pruebas, implementarlo y darle mantenimiento.

Preguntas del laboratorio 3

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

La recomendación es evitar generalmente el uso de !important, si se puede usar pero con moderación ya que puede dificultar el mantenimiento del código y generar conflictos con otras reglas de estilo.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Se debe escoger cuidadosamente una imagen de fondo porque puede afectar el rendimiento y la experiencia general del sitio web, ya que una mala elección puede afectar la legibilidad del contenido y la percepción del usuario.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

La recomendación es usar unidades relativas como % para diseños que puedan ser adaptables, mientras que px se puede usar para elementos específicos que requieren un tamaño fijo. Y por su parte se deberí evitar pt para pantallas ya que es una unidad de impresión y no se adapta bien a diferentes dispositivos.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

El uso de una versión minimizada del CSS mejora el rendimiento del sitio ya que se reduce el tamaño del archivo al eliminar aspectos inecesarios lo que a su vez disminuye el tiempo de carga y mejora la velocidad de navegación.

Preguntas del laboratorio 5

Describe Material Design

Material Design es un sistema de diseño desarrollado por Google que proporciona un conjunto de principios y componentes para crear interfaces de usuario atractivas, consistentes y funcionales en aplicaciones web y móviles. Yo considero que no es tan bueno ya que tiene una comunidad más pequeña que otro frameworks como Bootstrap, lo que puede dificultar encontrar recursos y soporte, además de que no es posible personalizarlo tanto como otros frameworks, lo que puede limitar la creatividad y flexibilidad en el diseño de las interfaces.
